JavaScript

数据类型

简单数据只有undefined, null, boolean, number, string（小写）

复杂数据只有object

函数类型：function

总结：

  1. undefined: typeof(undefined) -> undefined
  2. null: typeof(null) -> object，但null并非object，具有null值的变量也并非object
  3. boolean
  4. number: 特殊值：NaN, Infinity, NaN与任何数值进行运算的结果都是NaN，且NaN != NaN
  5. string

说明：

  • undefined, null, “”, 0转化为逻辑值时为false，除此之外的任何值转为逻辑值都为true； 且undefined, null, “”, 0, false中仅undefined == null成立，其余值互不相等
  • 完全由数字组成的字符串与对应的数的值是相等的，即"123" == 123，而"123" === 123 -> false（全等，数值和类型完全一致才返回true）
  • 所有简单类型都不是对象，JS没有将对象化的能力（仅object和function类型）赋予这些简单类型。直接被赋予简单类型常量值的标识符、变量和参数都不是对象

通过typeof获取变量类型：

var a = 5
console.log(typeof(a)) // number

// OR
console.log(typeof a) // number

类

  • object就是对象的类型。在JS中，不管多复杂的数据和代码，都可以组织成object形式的对象
  • JS中没有类的概念
var life = {}

for (life.age = 1; life.age <= 3; life.age ++) {
    switch (life.age) {
        case 1:
            life.body = "one";
            life.say = function(){console.log(this.age + this.body)};
            break;
        case 2:
            life.tail = "tail";
            life.gill = "gill";
            life.body = "two"
            life.say = function(){console.log(this.age + this.body + "-" + this.tail + ", " + this.gill)};
            break;
        case 3:
            delete life.tail;
            delete life.gill;
            life.legs = "four legs";
            life.lung = "lung";
            life.body = "frog";
            life.say = function() {console.log(this.age + this.body + "-" + this.legs + ", " + this.lung)};
            break;
    }
    life.say()
}
/*
1one
2two-tail, gill
3frog-four legs, lung
*/

函数

函数在JS中是function类型，当写下一个函数时，实际上是建立了function类型的一个实体

function myfunc() {
    console.log("mufunc1");
};

myfunc();
console.log(myfunc);
console.log(typeof(myfunc));

// 等价写法
var myfinc2 = function () {
    console.log("myfunc2");
}

myfinc2();
console.log(myfinc2);
console.log(typeof myfinc2);
/*
mufunc1
[Function: myfunc]
function
myfunc2
[Function: myfinc2]
function
*/

JavaScript执行引擎并非一行一行分析和执行程序，而是一段一段地分析和执行。

具体为：首先定义式的函数语句会被首先执行，函数定义执行完时候才会按顺序执行其他语句代码。

这一特性也被成为JS的预编译。此外，预编译的对象还包括所有的var变量的创建（初始值为undefined），以提高执行效率。

var myfunc = function () {
    console.log("mufunc");
};

myfunc();

var myfunc = function () {
    console.log("new myfunc");
}

myfunc();

function myfunction () {
    console.log("myfunction");
};

myfunction();

function myfunction () {
    console.log("new myfunction");
}

myfunction();
/*
mufunc
new myfunc
new myfunction
new myfunction // 结果同3
*/

JavaScript作用域

任何程序都会在一个原始的环境中运行，这个原始的环境称为全局环境。在JS中，这个全局环境就是一个对象，这个对象就是JS运行环境的跟。（对于在浏览器运行的JS来说，跟对象就是window对象） 对于全局的JS语句来说，window对象就是当前的作用域。

// 定义了window作用域下一个变量myName 
var myName = "test1";

// 定义window对象一个属性myName。
myName = "test2";

// 注：对于全局JS语句来说，加不加var无所谓，两种写法非常相似

待学习的知识点：

  • 函数对象有个caller属性，表示调用当前函数的上层函数。若为null，表示没被调用或是被全局代码调用
  • this
  • arguments
  • eval()
  • function具有的对象化的能力
  • function对象object对象的结合能力

对象

在JS中只有object和function有对象化的能力。 首先看function。

任何一个函数都可以为其动态或去除添加属性，这些属性可以是简单类型也可以是对象或其他函数，即函数具有对象的全部特征，函数完全可以当对象使用。

e.g. 下面的代码为函数动态添加了singer和content属性。

function sing() {
    console.log(sing.singer + " " + sing.content);
}

sing.singer = "甲";
sing.content = "abcd";
sing();

sing.singer = "乙";
sing.content = "efgh";
sing();
/*
甲 abcd
乙 efgh
*/

可以将对象当做数组，以属性名或方法名作为下标访问属性或调用方法

e.g.

var anObject = {};

anObject.aProperty = "property";
anObject.aFunction = function () {console.log("hello world");};

console.log(anObject["aProperty"]);
anObject["aFunction"]();

console.log(anObject.aProperty);
anObject.aFunction();

for (var s in anObject) {
    console.log(s + " " + typeof(anObject[s]));
}
/*
property
hello world
property
hello world
aproperty string
aFunction function
*/

function对象同理

var aFunction = function () {};

aFunction.aProperty = "property";
aFunction.aMethod = function () {console.log("hello world");};

console.log(aFunction["aProperty"]);
aFunction["aMethod"]();

console.log(aFunction.aProperty);
aFunction.aMethod();

for (var s in aFunction) {
    console.log(s + " " + typeof(aFunction[s]));
}
/*
property
hello world
property
hello world
aProperty string
aMethod function
*/
